Temp Pos Console(临时Pos机控制台系统)

主要功能：
	1.管理团购商家内容（类别，团购活动）
	2.生成打印商家兑换码表单（打印机出单）
	3.生成打印顾客兑换码小票（pos机出票）
	4.自定义表单系统
	5.自定义类别系统
	
	
程序定义：
	1.C/S桌面程序，使用java开发；
	2.应该使用SWT作为界面技术；
	

对象概念：

category(类别)：category代表着一类团购信息。
	每一个类别都对应着一个自定义表单，也就是说每一个类别的表单形式都不一定相同。
	
firm(商家、企业)：firm代表着一个商家信息。
	每一个商家都可能拥有若干团购信息
	
tuan(团购)：tuan代表着一个团购信息，或者叫活动信息
	每一个团购信息都必须属于某一个类别，创建一个团购过程根据不同的类别，也不尽相同。
	每一个团购信息也必须属于某一个商家
	团购信息必须包含开始时间和结束时间

key(兑换码)：key代表着一个兑换码
	兑换码的作用是用于兑换团购的实质内容
	一个团购可能包含多个兑换码，当然团购不一定都包含兑换码
	兑换码是在创建团购时手动创建好的一组指定数量的随即字符串，也有二维码形式
	**将来会开发一个手机应用，专门用于商家扫描二维码和随即字符串
	
	用户拿着兑换码，就可以前往商家处兑换响应的团购服务

	同样，面向商家的兑换码也是必须存在的，商家用来鉴别兑换码真伪

User(用户)：用户分为两级，管理员和操作员。
	操作员可以浏览团购信息，打印兑换码（商家表单和顾客小票）
	管理员可以管理操作员，并且执行系统的一切操作
